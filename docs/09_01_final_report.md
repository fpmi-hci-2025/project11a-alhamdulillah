# Финальный отчет по проекту

## 1. Постановка задачи

Целью проекта является разработка и проектирование информационной системы сети ресторанов, предназначенной для управления ресторанами, меню и позициями меню, а также предоставления API для веб- и мобильных клиентов.

В рамках работы необходимо:

* проанализировать предметную область и бизнес-процессы;
* спроектировать архитектуру системы;
* разработать UML-диаграммы;
* спроектировать схему базы данных.

---

## 2. Стратегия дизайна

При проектировании системы использованы следующие принципы:

* многоуровневая архитектура (API → бизнес-логика → доступ к данным);
* REST-подход для взаимодействия клиентов с сервером;
* модульность и расширяемость компонентов;
* разделение ответственности (SRP);
* использование ORM для отображения объектной модели на реляционную БД.
