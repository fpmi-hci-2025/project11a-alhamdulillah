# Анализ интерфейса по модели GOMS

## Введение

GOMS (Goals, Operators, Methods, Selection rules) - это модель для предсказания времени выполнения задач в интерфейсе. Модель разбивает задачу на элементарные операции и суммирует время их выполнения.

## 1. Модель KLM-GOMS

### 1.1. Временные константы операторов

| Оператор | Описание | Время (сек) |
|----------|----------|-------------|
| K | Keystroke (нажатие клавиши) | 0.2 |
| P | Pointing (наведение мыши) | 1.1 |
| H | Homing (перемещение руки мышь↔клавиатура) | 0.4 |
| M | Mental preparation (ментальная подготовка) | 1.2 |
| R | System response (ответ системы) | 1.0 |
| B | Button press (клик мыши) | 0.1 |
| T | Tab key press | 0.2 |

**Источник**: Card, Moran, & Newell (1983) - The Psychology of Human-Computer Interaction

### 4.3. Рекомендации по оптимизации

#### 4.3.1. Критические улучшения

1. **Обеспечить корректную Tab-навигацию**:
   - Логический порядок полей
   - Все интерактивные элементы доступны через Tab
   - Видимый фокус на текущем элементе

2. **Реализовать автозаполнение браузером**:
   - Атрибуты `autocomplete` для всех полей форм
   - `autocomplete="name"`, `autocomplete="email"` и т.д.

3. **Оптимизировать форму оформления заказа**:
   - Сделать поле "Комментарий" необязательным и менее заметным
   - Запоминать последний выбранный адрес

#### 4.3.2. Дополнительные улучшения

1. **Клавиатурные сокращения**:
   - `Alt+S` для поиска
   - `Alt+C` для корзины
   - `Ctrl+Enter` для подтверждения форм

2. **Умные дефолты**:
   - Предзаполнять часто используемые значения
   - Запоминать выбор способа доставки/оплаты

3. **Автофокус**:
   - Первое поле формы автоматически в фокусе
   - После ошибки - фокус на проблемное поле

4. **Inline-валидация**:
   - Проверка данных по мере ввода
   - Не блокирующая (позволяет продолжить ввод)
