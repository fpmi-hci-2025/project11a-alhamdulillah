# Обратная карточная сортировка (офлайн)

## Введение

Обратная карточная сортировка (Reverse Card Sorting) - метод тестирования информационной архитектуры, при котором пользователям показывают готовую структуру навигации и просят найти в ней заданные элементы. Это позволяет оценить, насколько интуитивно понятна предложенная структура.

## 1. Методология

### 1.1. Выбранный способ

Для проекта "Махачкала" используется **третий способ - тестирование экранов** с пронумерованными опциями. Этот метод выбран по следующим причинам:

1. Позволяет тестировать реальные экраны приложения
2. Легко воспроизвести условия использования
3. Дает количественные результаты (номер раздела)
4. Не требует сложной подготовки материалов

### 1.2. Подготовка материалов

#### Для веб-приложения

Подготовлены пронумерованные экраны следующих разделов:

1. **Главная страница** (верхнее меню)
   1. Главная
   2. Меню
   3. Рестораны
   4. Акции
   5. Мои заказы
   6. Профиль
   7. Корзина

2. **Страница меню** (категории)
   1. Все блюда
   2. Шаурма
   3. Напитки
   4. Закуски
   5. Десерты
   6. Соусы

3. **Страница профиля** (подразделы)
   1. Личные данные
   2. История заказов
   3. Бонусы и акции
   4. Адреса доставки
   5. Способы оплаты
   6. Настройки
   7. Выход

4. **Админ-панель** (главное меню)
   1. Управление меню
   2. Заказы
   3. Клиенты
   4. Сотрудники
   5. Аналитика
   6. Акции и промокоды
   7. Настройки
   8. Отчеты

#### Для мобильного приложения

Подготовлены пронумерованные экраны следующих разделов:

1. **Главный экран** (нижнее меню)
   1. Главная
   2. Меню
   3. Заказы
   4. Профиль

2. **Экран меню** (категории и фильтры)
   1. Все блюда
   2. Шаурма
   3. Напитки
   4. Закуски
   5. Фильтры
   6. Поиск

3. **Экран профиля** (опции)
   1. Редактировать профиль
   2. Мои заказы
   3. Бонусы
   4. Адреса
   5. Способы оплаты
   6. Настройки
   7. Поддержка
   8. Выход

### 5.2. Проблемные категории

**Веб-приложение:**

1. **Изменение номера телефона** (80% успеха):
   - **Проблема**: Один участник выбрал "Настройки" вместо "Личные данные"
   - **Причина**: Неоднозначность между "Личными данными" и "Настройками"
   - **Рекомендация**: Переименовать "Личные данные" в "Мой профиль" или объединить с "Настройками"

2. **Просмотр бонусов** (80% успеха):
   - **Проблема**: Один участник искал в разделе "Акции" на главной странице
   - **Причина**: Логическая связь "бонусы = акции"
   - **Рекомендация**: Добавить индикатор бонусов на главную страницу или дублировать в разделе "Акции"

3. **Добавление адреса доставки** (80% успеха):
   - **Проблема**: Один участник искал в разделе "Рестораны"
   - **Причина**: Связь адреса с локацией ресторанов
   - **Рекомендация**: Улучшить название "Адреса доставки" → "Мои адреса"

4. **Создание промокода** (80% успеха):
   - **Проблема**: Один участник выбрал раздел "Акции" вместо "Акции и промокоды"
   - **Причина**: Непонятно различие между разделами
   - **Рекомендация**: Уточнить названия разделов в админ-панели

**Мобильное приложение:**

1. **Поиск блюда по названию** (80% успеха):
   - **Проблема**: Один участник выбрал "Все блюда" вместо "Поиск"
   - **Причина**: Ожидание увидеть поле поиска на главном экране меню
   - **Рекомендация**: Сделать поиск более заметным или добавить на главный экран меню

2. **Фильтр вегетарианских блюд** (80% успеха):
   - **Проблема**: Один участник выбрал только "Меню" без подраздела
   - **Причина**: Ожидание увидеть фильтры сразу в меню
   - **Рекомендация**: Сделать фильтры более заметными в интерфейсе меню

3. **Связь с поддержкой** (80% успеха):
   - **Проблема**: Один участник выбрал раздел "Заказы" вместо "Профиль → Поддержка"
   - **Причина**: Связывание проблем с заказами с разделом заказов
   - **Рекомендация**: Добавить кнопку "Помощь" в раздел "Заказы" или на главный экран
