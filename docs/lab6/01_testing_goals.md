# Задачи тестирования и численные оценки качества

## 1. Определение задач тестирования

### 1.1. Задачи usability-тестирования

#### Для веб-приложения

1. **Оценка эффективности навигации**
   - Измерить время, необходимое пользователю для нахождения нужного пункта меню
   - Оценить количество кликов для выполнения основных задач
   - Выявить проблемные зоны навигации

2. **Оценка эффективности оформления заказа**
   - Измерить время оформления заказа от выбора блюда до подтверждения
   - Оценить количество ошибок при заполнении форм
   - Выявить барьеры в процессе оформления заказа

3. **Оценка понятности интерфейса**
   - Проверить понимание пользователями терминологии
   - Оценить ожидания от поведения элементов интерфейса
   - Выявить несоответствия между ожиданиями и реальным поведением

#### Для мобильного приложения

1. **Оценка удобства жестового управления**
   - Измерить время выполнения основных жестов (тап, свайп, скролл)
   - Оценить эргономику размещения элементов для одноручного использования
   - Выявить проблемы с размером сенсорных зон

2. **Оценка скорости выполнения задач**
   - Измерить время быстрого заказа "шаурмы за 5 рублей"
   - Оценить скорость поиска ближайшего ресторана
   - Измерить время отслеживания статуса заказа

3. **Оценка мобильной навигации**
   - Проверить понятность структуры меню
   - Оценить удобство переключения между разделами
   - Выявить проблемы с возвратом к предыдущим экранам

### 1.2. Целевые пользователи для тестирования

На основе [профилей пользователей](../research/USERS.md) определены следующие группы для тестирования:

1. **Студенты (20-25 лет)** - основная целевая аудитория
   - Уровень технической грамотности: высокий
   - Частота использования: 3-5 раз в неделю
   - Основная мотивация: скорость и низкая цена

2. **Гастарбайтеры (25-35 лет)** - вторая целевая группа
   - Уровень технической грамотности: средний
   - Частота использования: ежедневно
   - Основная мотивация: привычная еда и доступная цена

3. **Скуфы (35-50 лет)** - дополнительная группа
   - Уровень технической грамотности: средний
   - Частота использования: 2-3 раза в неделю
   - Основная мотивация: низкая цена и большие порции

### 1.3. Тестовые сценарии

#### Сценарий 1: Быстрый заказ акционной шаурмы (веб и мобильное)

**Описание:** Артем (студент) хочет быстро заказать шаурму за 5 рублей между парами.

**Шаги:**
1. Открыть приложение
2. Найти акционное предложение "Шаурма за 5 рублей"
3. Добавить в корзину
4. Выбрать способ получения (самовывоз)
5. Выбрать ближайший ресторан
6. Выбрать способ оплаты (картой онлайн)
7. Подтвердить заказ

**Критерии успеха:**
- Время выполнения: не более 2 минут
- Количество кликов/тапов: не более 10
- Отсутствие ошибок при оформлении

#### Сценарий 2: Поиск ближайшего ресторана (веб и мобильное)

**Описание:** Алексей (турист) хочет найти ближайший ресторан "Махачкала" в незнакомом городе.

**Шаги:**
1. Открыть раздел "Рестораны" или "Карта"
2. Разрешить доступ к геолокации
3. Посмотреть список/карту с ближайшими ресторанами
4. Выбрать ближайший ресторан
5. Посмотреть адрес и время работы

**Критерии успеха:**
- Время выполнения: не более 1 минуты
- Количество кликов/тапов: не более 5
- Понятность отображения расстояния и маршрута

#### Сценарий 3: Просмотр меню и добавление нескольких позиций (веб и мобильное)

**Описание:** Алихан (гастарбайтер) хочет заказать несколько блюд для бригады.

**Шаги:**
1. Открыть раздел "Меню"
2. Просмотреть категории блюд
3. Добавить в корзину: 3 шаурмы за 5 рублей
4. Добавить в корзину: 2 шаурмы с курицей
5. Добавить в корзину: 5 напитков
6. Перейти в корзину
7. Проверить состав заказа
8. Оформить заказ с доставкой

**Критерии успеха:**
- Время выполнения: не более 4 минут
- Отсутствие ошибок в составе заказа
- Понятность процесса изменения количества

#### Сценарий 4: Отслеживание статуса заказа (веб и мобильное)

**Описание:** Артем оформил заказ и хочет узнать, когда он будет готов.

**Шаги:**
1. Открыть раздел "Мои заказы" или "Профиль"
2. Найти текущий активный заказ
3. Посмотреть статус заказа
4. Получить информацию о времени готовности

**Критерии успеха:**
- Время выполнения: не более 30 секунд
- Количество кликов/тапов: не более 3
- Понятность отображения статуса

#### Сценарий 5: Управление меню (веб, администратор)

**Описание:** Администратор ресторана хочет добавить новое блюдо в меню.

**Шаги:**
1. Войти в админ-панель
2. Перейти в раздел "Управление меню"
3. Нажать "Добавить новое блюдо"
4. Заполнить форму: название, описание, цена, категория
5. Загрузить фото блюда
6. Сохранить изменения

**Критерии успеха:**
- Время выполнения: не более 3 минут
- Отсутствие ошибок валидации
- Немедленное отображение в меню

## 3. Критерии качества интерфейса

### 3.1. Общие критерии

1. **Эффективность**: Пользователь должен выполнить задачу с минимальным количеством действий
2. **Результативность**: Задача должна быть выполнена успешно не менее чем в 85% случаев
3. **Удовлетворенность**: Пользователи должны оценить интерфейс не ниже 4.0 из 5

### 3.2. Критерии для веб-приложения

1. **Навигация**: Не более 3 кликов до любого раздела меню
2. **Скорость загрузки**: Все экраны загружаются за 2 секунды
3. **Адаптивность**: Корректное отображение на разрешениях от 1280x720 до 1920x1080
4. **Доступность**: Соответствие WCAG 2.1 уровня AA

### 3.3. Критерии для мобильного приложения

1. **Размер сенсорных зон**: Не менее 44x44 пикселя (iOS) / 48x48 dp (Android)
2. **Одноручное использование**: Основные элементы в зоне досягаемости большого пальца
3. **Жестовое управление**: Поддержка стандартных жестов (свайп назад, pull-to-refresh)
4. **Производительность**: Отклик на тап не более 100 мс

## 4. План тестирования

### 4.1. Количество участников

- **Веб-приложение**: 6 пользователей (по 2 из каждой целевой группы)
- **Мобильное приложение**: 6 пользователей (по 2 из каждой целевой группы)

### 4.2. Формат тестирования

1. **Обратная карточная сортировка**: 5 участников для каждого метода
2. **Тестирование ожиданий**: 5 участников
3. **Task Success / Time-on-Task**: 6 участников
4. **SUS анкета**: все участники тестирования (6 человек)

### 4.3. График тестирования

| Этап | Длительность | Участники |
|------|--------------|-----------|
| Обратная карточная сортировка (offline) | 2 дня | 5 человек |
| Обратная карточная сортировка (TreeJack) | 3 дня | 5 человек |
| Тестирование ожиданий | 2 дня | 5 человек |
| Task Success / Time-on-Task | 3 дня | 6 человек |
| SUS анкета | 1 день | 6 человек |
| **Итого** | **11 дней** | - |

## 5. Инструменты для тестирования

1. **Карточная сортировка**:
   - Offline: карточки, таймер, форма для записи результатов
   - Online: TreeJack от OptimalWorkshop

2. **Измерение времени**: секундомер / встроенный таймер в инструменты

3. **SUS анкета**: Google Forms / бумажная форма

4. **GOMS/KLM расчеты**: Excel-таблицы с формулами

5. **Запись тестирования**: 
   - Веб: screen recording (OBS Studio)
   - Мобильное: screen recording (встроенный в ОС)
