# Обратная карточная сортировка с TreeJack

## Введение

TreeJack - онлайн-инструмент от Optimal Workshop для тестирования информационной архитектуры. В отличие от офлайн-метода, TreeJack позволяет проводить тестирование удаленно, собирать статистику автоматически и анализировать пути пользователей через дерево навигации.

## 1. Подготовка тестирования

### 1.1. Инструмент TreeJack

**Официальная документация**: https://www.optimalworkshop.com/treejack

**Основные возможности**:
- Создание древовидной структуры сайта/приложения
- Формулирование задач для участников
- Автоматический сбор статистики
- Анализ успешности и путей навигации
- Экспорт результатов

**Веб-приложение**: 8 участников

1. **Welcome screen**:
   > "Добро пожаловать в тестирование информационной архитектуры приложения 'Махачкала'! Вам будут даны задачи, для выполнения которых нужно найти правильный раздел в структуре приложения. Кликайте по разделам, чтобы раскрыть подразделы. Когда найдете нужный раздел, нажмите 'I'd find it here'. Если не можете найти, нажмите 'I'd give up'."

2. **Randomize tasks**: Да (для каждого участника свой порядок)

3. **Skip logic**: Нет

4. **Time limit**: Нет (но фиксируется время)

## 5. Анализ результатов

### 5.1. Сравнение с офлайн-методом

| Метрика | Офлайн (веб) | TreeJack (веб) | Офлайн (моб) | TreeJack (моб) |
|---------|--------------|----------------|--------------|----------------|
| Success Rate | 90% | 81.7% | 94% | 88.3% |
| Среднее время | 4.6 сек | 12.3 сек | 3.7 сек | 9.8 сек |

**Причины расхождений:**
1. TreeJack включает более сложные многоуровневые задачи
2. Участники не видят всю структуру сразу (нужно раскрывать разделы)
3. Нет возможности получить подсказку от модератора
4. Более реалистичное моделирование использования

### 5.2. Проблемные задачи (веб)

#### Задача 12: История бонусов (62.5% успеха)

**Проблема**: 
- 2 участника искали в "Акции → Программа лояльности"
- 1 участник искал в "Мои заказы → История заказов"

**Причина**: 
- Связь бонусов с акциями
- Ассоциация истории бонусов с историей заказов

**Пути навигации**:
```
Правильный путь (5 участников):
Профиль → Бонусы и акции → История начислений

Неправильные пути:
1. Акции → Программа лояльности (2 участника)
2. Мои заказы → История заказов (1 участник)
```

**Рекомендация**: 
- Дублировать информацию о бонусах в разделе "Акции"
- Добавить ссылку на историю бонусов из истории заказов

#### Задача 13: Карта по умолчанию (62.5% успеха)

**Проблема**:
- 2 участника искали в "Корзина → Оформить заказ"
- 1 участник искал в "Профиль → Личные данные"

**Причина**:
- Связь способа оплаты с процессом оформления заказа
- Восприятие платежной информации как личных данных

**Пути навигации**:
```
Правильный путь (5 участников):
Профиль → Способы оплаты → Способ оплаты по умолчанию

Неправильные пути:
1. Корзина → Оформить заказ (2 участника)
2. Профиль → Личные данные (1 участник)
```

**Рекомендация**:
- Добавить возможность изменить способ оплаты по умолчанию при оформлении заказа
- Переименовать "Личные данные" в "Персональная информация" для более четкого разделения

#### Задача 11: Изменить пароль (75% успеха)

**Проблема**:
- 2 участника искали в "Профиль → Личные данные"

**Причина**:
- Восприятие пароля как части личных данных

**Рекомендация**:
- Добавить подраздел "Безопасность" или "Пароль и безопасность" в "Настройки"
- Дублировать ссылку на изменение пароля в "Личных данных"

### 5.3. Проблемные задачи (мобильное)

#### Задача 11: Правила программы лояльности (75% успеха)

**Проблема**:
- 2 участника искали в "Главная → Акции"

**Причина**:
- Логическая связь программы лояльности с акциями

**Рекомендация**:
- Добавить информацию о программе лояльности на главный экран в разделе "Акции"
- Добавить ссылку из "Акции" в "Профиль → Бонусы"

#### Задача 15: История бонусов (75% успеха)

**Проблема**:
- 1 участник искал в "Заказы → История"
- 1 участник отказался от задачи

**Причина**:
- Связь истории бонусов с историей заказов
- Глубокая вложенность раздела

**Рекомендация**:
- Показывать историю бонусов на главном экране раздела "Бонусы"
- Добавить индикатор недавних начислений
